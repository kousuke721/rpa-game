📄 01 HTMLファイルの追加手順
1
HTMLファイルをrpa-gameフォルダに入れる
Finderでデスクトップ → rpa-game フォルダにドラッグ＆ドロップ
2
ターミナルでrpa-gameに移動
cd ~/Desktop/rpa-game
3
変更をステージング・コミット・プッシュ
git add . git commit -m "新しいページを追加" git push
4
スマホからアクセス確認
数分後に以下のURLでアクセス可能になる
https://kousuke721.github.io/rpa-game/ファイル名.html
💡 ファイルを上書きしたいときも同じ手順でOK。git が差分を自動で検出してくれる。
⌨️ 02 よく使うGitコマンド
📋 状態確認・基本操作 毎日使う
# 変更状態を確認（まず確認する習慣を） git status # 全ファイルをステージングに追加 git add . # 特定ファイルだけ追加したい場合 git add ファイル名.html # コミット（変更を記録） git commit -m "変更内容のメモ" # GitHubにアップロード git push # GitHubから最新を取得（他の端末で変更した後） git pull
🕐 履歴・差分確認 たまに使う
# コミット履歴を確認 git log --oneline # 変更した内容の差分を確認 git diff # 直前のコミットを取り消す（ファイルは残る） git reset --soft HEAD~1
🔙 ファイルを元に戻す 慎重に使う
# 特定ファイルの変更を取り消す（保存前の状態に戻す） git checkout -- ファイル名.html # ステージングを取り消す（git addを取り消す） git reset HEAD ファイル名.html
⚠️ checkout で元に戻すと変更内容は完全に消えるので注意。
🌿 03 ブランチの基本
ブランチとは「作業用のコピー」です。mainブランチを壊さずに新機能を試せます。
main ●━━━━━━━━━━━━━●━━━● ↘ ↗ dev ●━━━━━●
# 現在のブランチを確認 git branch # 新しいブランチを作って移動 git checkout -b dev # mainブランチに戻る git checkout main # devブランチをmainに統合 git merge dev # 不要なブランチを削除 git branch -d dev
💡 今は一人運用なのでブランチは必須ではありません。「新機能を試したい」「壊したくない」ときに使い始めるとよいです。

🚨 05 よくあるエラーと解決法
Host key verification failed.
SSHの接続先が未登録のため発生。初回接続時によく出る。
解決法
ssh-keyscan github.com >> ~/.ssh/known_hosts ssh -T git@github.com
error: failed to push some refs
GitHubのリモートが手元より新しい状態。GitHub上で直接編集した後によく出る。
解決法
# まずpullしてから再度push git pull git push
nothing to commit, working tree clean
エラーではなく「変更がない」という意味。commitするものがない状態。
対応
ファイルを編集・保存してから git add . をやり直す。
Permission denied (publickey)
SSHキーがGitHubに登録されていないか、接続設定がおかしい。
解決法
# SSH接続を確認 ssh -T git@github.com # 公開鍵をコピーしてGitHubに再登録 pbcopy < ~/.ssh/id_ed25519.pub
zsh: command not found: code
VS CodeのCLIコマンドが有効になっていない。
解決法
VS Codeで ⌘+⇧+P →「Shell Command: Install 'code' command in PATH」を実行してターミナルを再起動。
🌐 06 GitHub Pages の確認・更新
🔗 自分のページURL一覧
# ベースURL https://kousuke721.github.io/rpa-game/ # 各ファイル https://kousuke721.github.io/rpa-game/index.html https://kousuke721.github.io/rpa-game/rpa-academy.html https://kousuke721.github.io/rpa-game/git-guide.html
⏱️ pushしたのに反映されない よくある
GitHub Pagesはpush後、反映まで1〜3分かかります。
1
GitHubのActionsタブで進捗確認
リポジトリ → Actions → 最新の実行が✅になれば反映済み
2
キャッシュをクリアして確認
Mac: ⌘+⇧+R　スマホ: ブラウザを閉じて再度開く
📌 GitHub Pages の設定確認：リポジトリ → Settings → Pages → Branch: main / /(root) になっていればOK
